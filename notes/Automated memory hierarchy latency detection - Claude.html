The aim of this project is to build an automated Python framework — called "Auto-Echo" — that can discover the structure of a computer's memory hierarchy purely from latency measurements, without any prior knowledge of the system. Rather than relying on manual inspection of graphs or pre-existing hardware specifications, the framework collects raw memory access timing data using an echolocation-style probe, cleans it with statistical filtering, and then applies unsupervised machine learning algorithms (such as K-Means, Gaussian Mixture Models, or DBSCAN) to automatically identify distinct latency clusters. The goal is for the system to independently determine how many memory levels exist on a given machine and what the latency characteristics of each level are.

The broader purpose is to turn what has traditionally been a manual, expert-driven process of "reading the bumps" in latency plots into a fully automated, reproducible analysis pipeline. By incorporating model selection techniques like the Elbow Method or Silhouette Score, the framework can self-determine the optimal number of clusters and produce a structured report mapping each cluster to a likely memory level (e.g. L1 cache, L2, L3, DRAM). The project is ultimately validated by checking whether its automated findings match known hardware specifications, making it a practical fusion of systems programming and applied machine learning — moving the field from visual pattern-spotting toward algorithmic discovery.

Based on the project description, the primary objectives are:

1. Automated Memory Hierarchy Discovery To build a framework that can identify the number and boundaries of memory hierarchy levels (L1, L2, L3, DRAM) on any target machine without needing prior knowledge of its specifications — replacing manual inspection with algorithmic detection.

2. Latency Data Collection & Cleaning To implement (or reuse) an echolocation-style memory probe that gathers a large dataset of memory access latencies, then apply statistical noise filtering (outlier removal, moving averages) to produce clean, reliable timing data suitable for analysis.

3. Unsupervised Machine Learning Application To apply clustering algorithms — specifically K-Means, Gaussian Mixture Models, or DBSCAN — to the latency data so that distinct memory levels emerge naturally as separate clusters, without any labelled training data.

4. Automatic Model Selection To implement a principled method (Elbow Method or Silhouette Score) that automatically determines the most likely number of memory levels, removing the need for a human to decide how many clusters to look for.

5. Validation Against Known Hardware To verify the framework's accuracy by comparing its automated findings against the documented specifications of one or two known test machines — confirming it correctly identifies the expected memory hierarchy levels.

6. Reporting To produce a structured, readable output report that presents the discovered number of memory levels and the latency range associated with each one.
